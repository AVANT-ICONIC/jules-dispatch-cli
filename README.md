## Category Focus

- **Jules Dispatch = an agent-first CLI** — Stable JSON-native interface, zero-touch session management, and PR orchestration are what Jules Dispatch delivers. Built for AI agents and power users who live in the terminal.
- **Not Jules Dispatch** = a web dashboard, a Jules replacement, or a general-purpose automation tool. It wraps the Jules API — it doesn't reinvent it.

---

## Overview

Jules Dispatch is a specialized CLI that exposes the full [Google Jules](https://jules.google.com) API through a terminal-native interface. Every command supports `--json` output — clean, typed, and stable — so AI agents (Claude Code, Codex, Gemini, etc.) can dispatch jobs, monitor sessions, and handle PRs without touching a browser.

```bash
# Dispatch a new task to Jules
jules sessions create --repo owner/repo --prompt "Refactor the auth middleware to use JWT"

# Monitor progress (JSON-native for easy parsing)
jules sessions get SESS_ID --json

# List active PRs generated by Jules
jules prs list --repo owner/repo
```

```bash
# Plan review mode — Jules pauses after generating a plan
jules sessions create --repo owner/repo \
  --prompt "Migrate all API routes to the new error handling pattern" \
  --approve-plan

# Review and approve when ready
jules sessions approve SESS_ID
```

## Why Not the Web UI?

| Capability | Jules Web UI | Jules Dispatch CLI |
| :--- | :---: | :---: |
| Dispatch jobs | ✓ | ✓ |
| Agent-readable output | — | ✓ Stable JSON on every command |
| Scriptable / automatable | — | ✓ Shell pipelines, CI/CD, agent loops |
| PR lifecycle management | Manual | ✓ List, view, merge from terminal |
| Session monitoring (polling) | Refresh browser | ✓ `--json` + `jq` for real-time checks |
| Offline / batch workflows | — | ✓ Queue jobs, check later |
| Integration with gh CLI | — | ✓ Direct PR merge, comment, view |

## Setup

### Prerequisites

- [Bun](https://bun.sh) v1.0+
- [gh CLI](https://cli.github.com) authenticated (`gh auth login`)
- [Google Jules](https://jules.google.com) account with the GitHub App installed
- Jules API key from **Settings > API Keys**

### Installation

```bash
git clone https://github.com/AVANT-ICONIC/jules-dispatch.git
cd jules-dispatch
bun install
```

### Configuration

Create a `.env` file in the project root:

```env
JULES_API_KEY=your_key_here
```

The CLI reads `JULES_API_KEY` from the environment. No other configuration is required. If the variable is missing, the CLI exits with a clear error message.

```bash
# Verify everything works
bun run src/index.ts sources list --json
```

## Commands

| Command | Description |
| :--- | :--- |
| `jules sources list` | List all repositories connected to Jules |
| `jules sessions create` | Start a new coding job with a prompt |
| `jules sessions get` | Get detailed state and recent activities for a session |
| `jules sessions list` | List sessions, filterable by repo and state |
| `jules sessions reply` | Send a message to Jules when it asks a question |
| `jules sessions approve` | Approve Jules's plan and let it execute |
| `jules sessions activities` | Show the activity log for a session |
| `jules prs list` | List PRs generated by Jules (cross-references sessions) |
| `jules prs view` | View details for a specific PR |
| `jules prs merge` | Merge a Jules PR |
| `jules prs comment` | Post a comment on a Jules PR |

Every command accepts `--json` for machine-readable output. Run `jules --help` for full usage details, or see the [Command Reference](./docs/command-reference.md).

## For AI Agents

AI agents should **always** use the `--json` flag. This ensures the output is a stable JSON object that can be reliably parsed and acted upon — no human-readable formatting, no ANSI escape codes, no surprises.

```bash
# Dispatch a job and capture the session ID
SESSION_ID=$(jules sessions create \
  --repo "owner/repo" \
  --prompt "fix: typo in README" \
  --json | jq -r '.id')

# Poll until complete
while true; do
  STATE=$(jules sessions get "$SESSION_ID" --json | jq -r '.session.state')
  if [ "$STATE" = "COMPLETED" ] || [ "$STATE" = "FAILED" ]; then
    break
  fi
  sleep 30
done

# Find and merge the PR
jules prs list --repo "owner/repo" --json | jq '.[] | select(.julesSessionId == "'"$SESSION_ID"'")'
```

See [AGENTS.md](./AGENTS.md) for the full agent integration guide with polling strategies, error handling, and state machine documentation.

### Session States

| State | Meaning | Agent Action |
| :--- | :--- | :--- |
| `IN_PROGRESS` | Jules is working | Poll again in 30s |
| `WAITING_FOR_INPUT` | Jules has a question | Read activities, reply |
| `PLAN_READY` | Jules generated a plan | Review and approve |
| `COMPLETED` | Jules finished | Check outputs for PR |
| `FAILED` | Jules failed | Report to user |

## Architecture

```
┌──────────────────────────────────────────────────────────┐
│                    Jules Dispatch CLI                     │
│                                                          │
│  ┌──────────────┐  ┌──────────────┐  ┌────────────────┐ │
│  │  Command Bus │  │  Jules API   │  │  Output Layer  │ │
│  │  (Commander) │  │  Client      │  │  (JSON/Human)  │ │
│  └──────┬───────┘  └──────┬───────┘  └───────┬────────┘ │
│         │                 │                   │          │
│         ▼                 ▼                   ▼          │
│  ┌──────────────┐  ┌──────────────┐  ┌────────────────┐ │
│  │  gh Bridge   │  │  Retry/429   │  │  Error Handler │ │
│  │  (PR mgmt)   │  │  Backoff     │  │  (typed errors)│ │
│  └──────────────┘  └──────────────┘  └────────────────┘ │
│                                                          │
│  Jules API (v1alpha) ──► https://jules.googleapis.com    │
└──────────────────────────────────────────────────────────┘
```

- **Command Bus**: [Commander.js](https://github.com/tj/commander.js) with subcommands, required options, and `--json` on every route.
- **Jules API Client**: Typed fetch wrapper with `X-Goog-Api-Key` auth, flat payload convention, and exponential backoff on 429 rate limits.
- **gh Bridge**: Swappable shell executor (`setExecGh` for testing) that wraps `gh pr list|view|merge|comment` with pre-flight auth checks.
- **Output Layer**: `printJson` for agents, `printHuman` for humans — zero global state, explicit mode per call.
